Introduction

This is a Technical Documentation for HTML and CSS. This page covers enough of the basics of HTML and CSS to get you started with making your own website. HTML is a markup language that is used to describe the general information and layout of the page. Basically a text document to say what should be written on the webpage. CSS stands for cascading style sheet. This is used to produce the layout and design of the page. This is all you need to make a basic webpage.

HTML Structural tags

Structural tags are containers that hold sections of your webpage. These sections can be referred to in CSS to style each section individually. Structural tags order your webpage and help with accessability as well. The webpages you build should be accessible to all, including those who may be using screen readers and those who have sight problems. More information can be found on accessibility.

When writing html, you use tags(<>) which have to be closed(</>). For example to write the title of the page you would use:
<title>Hello World!</title>
However, some tags are self inclosing meaning you do no have to close the tag using</tag name> An example of this would be something like the <br> which creates a new line.
SC will refer to self closing tags.

<!Doctype html> SC. HTML code must include this. It declares that this is a HTML document.
<html> The root element of a HTML page.
<head> Contains meta information about the HTML page. This contains information you do not want your user to see.
<title> Title of the page. This can be either be in the head or body tag. This is what is shown in the browser tab or title.
<body> This contains all all the html code that will be used in the markup.
<main> Main is the main information on the page. It does not include things like navigation bars headers and footers.
<h1> header 1 is the title for the first section of the page. each section can be titled with h1.
<h2> h2 is a subsection of the h1. you can have subsection of h2 with the h3 tags. this carries on all the way to h6.
<p> This is the paragraph tag.

HTML Hyperlink

<a> The anchor tag is used for hyperlinks. whatever is in between the open and close anchor tags is hyperlinked.
href The href attribute is where the url for hyperlink is put. Attributes are settings that go inside the tags that add functionality. The href accepts not only urls, but special values that you use for extra functionality. for example:
<a href="www.duck.com">goes to duck.com</a>
<a href="#">dead link</a>
<a href="id">scrolls to location of id on this webpage.</a>

You can make the link open a new tab using target="_blank.
Example:
<a href="www.duck.com" target="_blank">opens duck.com in new tab</a>

HTML Form

the <form> tag is container in which you put your input fields in. This can have the attribute action='url'. When the form is completed the information in the form will be sent to the url. This link should be a link to a form processor.

There are many types of inputs that you can put in a form. these are not attributes. they are tags. the go in between the form open tag and form close tag.
<input type="text" placeholder="faded text inside the input field"> SC. This will create a text box to type in. The placeholder is the text that shows up as the default text inside the input field.
<button type ="submit">text on button<button> This creates a button.

<label for="label(same as text)"> <input id="id" type="radio" name="of all radio buttons" value="value">text </label>
Here two tags are shown at the same time. The label tag wraps around the input tag. The label tag labels the input tag so screen readers can understand what the input is saying. This is also the text that shows up next to the radio. On the input tag, we have the type radio. This gives a select option, similar to checkbox except only one can be picked at a time. your would use this for something like picking a gender. The name attribute is what links the radios together. The value attribute is what is sent to the form processor.

<input id="id" type="checkbox" name="personality" checked> This is a check box input field. Again, name is what connects all the checkboxes. The checked attribute is optional and checks inputs by default.

CSS Basics

To style you must first select what you want to style. This is done by the name of what you want to style followed by {}. Inside the curly brackets{} is the code that styles the css tag. For example, if you wanted to change the background color of the webpage you would use the following:
body {background-color:red;}

to reference a structural tag just use the name of the tag. To reference an id use #id. To reference a tag use .class.

CSS goes in between the style tag (<style>) inside the html file. The html file is normally named index.html. If you want to keep the css in a different file, you can just add a link to the css file using
<link href="url" rel="stylesheet" type="text/css">

You can have variables in CSS. They are made using the following:
var(--variablename)
To use the variable use: --variablename

CSS Hierarchy

When formatting your CSS you will probably start overlapping functions. When this happens CSS will apply the function with the greatest importance. The order below is from most important to least.

  1. Important attribute: color:red !important;
  2. Inline HTML attribute: style="color:orange;"
  3. ID: #id {color:yellow;}
  4. Classes: .class{color:green;}
  5. Elements: h1{color:blue;}

CSS Media query

Media query's are applied when the size of the display matched the statement. In this example the media query changes the size of the variable inside the global tag,root if the screen size is smaller then 350px.
@media (max-width: 350px) {
:root {
--varible:200px;}
}

CSS Pseudo-classes

Pseudo-classes only apply if certain events are triggers. some examples are:
a:hover{}
a::before {
content: "";
a::after {
content: ""}
 These pseudo-classes are being applied to an anchor. The hover pseudo-class applies if the mouse if hovering over the anchor. The ::before tag applies it before the anchor. And vice versa for the ::after.

Accessibility

Here are some key points to remember when coding for accessibility.

  • Have well organized code that uses appropriate markup.
  • Ensure all non text content such as images and audio have an alt attribute that clearly describes the content. Make sure you done use special characters.
  • Create an easily navigated page that has keyboard friendly navigation.
  • Images that are decorative should have an empty alt.
  • When picking colors, make sure there is high enough contrast for the visually impaired.
  • The heading tags, h1, h2... should be used correctly with h2 headings being subsection of h1's.
  • Include meta information inside head.
